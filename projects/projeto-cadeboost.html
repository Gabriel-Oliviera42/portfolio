Projeto Cadeboost

Este projeto foi um marco fundamental na minha evolução técnica. Trata-se do desenvolvimento completo de uma Landing Page moderna com o objetivo de dominar o alinhamento avançado de layouts, a organização estrutural do CSS e interações dinâmicas nativas.

Por que eu fiz

Depois dos meus primeiros contatos com programação web, percebi que precisava melhorar bastante a parte visual dos meus projetos. Na época, eu conseguia fazer as funcionalidades funcionarem, mas o resultado final deixava muito a desejar. Foi nesse período que participei de uma apresentação em que um dos palestrantes era o Will, criador de conteúdo focado em front-end no YouTube.

Acompanhar os vídeos dele me ajudou a entender melhor como um projeto visual é construído. Muitos dos conteúdos eram desenvolvimentos guiados: ele mostrava o layout no Figma e explicava como transformá-lo em código. Hoje isso pode parecer algo simples, mas, para alguém que aprendia tudo por tentativa e erro, foi uma grande mudança de perspectiva.

Esse projeto acabou sendo um marco para mim. Não porque passei a criar interfaces incríveis da noite para o dia, mas porque comecei a entender melhor o processo de desenvolvimento. Aprendi a separar responsabilidades, saber o que resolver com HTML, o que deixar para o CSS e como organizar melhor cada etapa da construção de um site. Foi um aprendizado que influenciou diretamente a forma como desenvolvo projetos até hoje.

Meu processo

  1. Planejamento e HTML Semântico. Antes de qualquer estilização, analisei a interface para estruturar as tags corretas. Aprendi a dividir a landing page em blocos lógicos usando header, main, sections e footer, garantindo que o código ficasse acessível, limpo e bem interpretado pelos motores de busca.
  2. Arquitetura CSS e Layouts Fluidos. Construí os estilos focando na reutilização de classes e na criação de layouts modernos com Flexbox e CSS Grid. O maior desafio foi garantir que os elementos mantivessem o alinhamento harmônico e as proporções visuais corretas à medida que a tela diminuía.
  3. Interatividade com JavaScript Vanilla. Implementei as funções dinâmicas da página manipulando diretamente o DOM (Document Object Model). Desenvolvi seleções de elementos, escutadores de eventos (event listeners) e alternâncias de estados que trouxeram vida à interface através de animações controladas e menus interativos.
  4. Resolução de Bugs e Testes de Responsividade. Dediquei uma parte massiva do tempo utilizando as ferramentas de desenvolvedor (DevTools) para inspecionar quebras de layout. Ajustei minuciosamente os breakpoints por meio de Media Queries para assegurar que a experiência de navegação fosse impecável em smartphones, tablets e desktops.

Usando este projeto

Esse projeto tem uma característica interessante. Por ser uma landing page simples, sem navegação complexa e com uma estrutura bem organizada, ele continuou sendo útil mesmo depois de concluído. Durante minha Iniciação Científica sobre acessibilidade web, utilizei este site como um dos exemplos nas apresentações e demonstrações práticas. Você pode conferir mais detalhes sobre esse trabalho na página da Iniciação Científica.

Além disso, utilizei este projeto para demonstrar a importância de escrever um código bem estruturado e acessível. Para isso, comparei duas versões de um mesmo site: esta versão, que possui alguns problemas de acessibilidade mais sutis, e outra versão. criada propositalmente com diversos erros. O objetivo era mostrar como barreiras digitais podem afetar a experiência de navegação de diferentes usuários e como muitas dessas falhas passam despercebidas por quem não depende de recursos de acessibilidade no dia a dia.

O que esse projeto me ensinou

Organização é tudo

Entendi que um código bem estruturado economiza horas de manutenção. Separar responsabilidades no CSS e adotar boas práticas de nomenclatura evita conflitos de estilização em páginas grandes.

Poder do CSS Nativo

Descobri que Flexbox e CSS Grid combinados são extremamente poderosos. Entender como e quando utilizar cada um deles resolve praticamente qualquer desafio complexo de posicionamento na web.

Manipulação consciente do DOM

Aprender JavaScript puro (Vanilla) para gerenciar estados e interações me deu uma base sólida. Agora compreendo exatamente o que acontece nos bastidores quando uma interface reage ao usuário.

Olhar Clínico de Interface

O projeto refinou meu senso de UI. Aprendi a valorizar o espaçamento (padding/margin), o alinhamento rigoroso dos elementos e a consistência tipográfica para construir produtos visualmente atraentes.

Vídeo usado como referência

O projeto foi inteiramente baseado na aula especial do canal Codeboost, focada em construir uma landing page profissional do zero absoluto. Utilizei este conteúdo como mentor virtual para compreender as decisões de desenvolvimento adotadas pelo instrutor Will durante a criação do projeto.

Conclusões

Olhando para trás, o projeto Cadeboost representa um momento crucial da minha jornada de aprendizado. Ele simboliza a transição exata de quando deixei de ser apenas um estudante passivo que assistia tutoriais e passei a ser um desenvolvedor ativo, capaz de enfrentar linhas de código longas, gerenciar centenas de regras CSS e debugar problemas complexos de renderização de forma consciente.

A experiência adquirida aqui mudou completamente minha velocidade de entrega e meu nível de exigência técnica. O tempo considerável que passei quebrando a cabeça para ajustar pequenos detalhes finos me deu a resiliência necessária para encarar arquiteturas muito maiores e tecnologias mais avançadas (como frameworks SPA e pré-processadores) com total segurança.

Por fim, considero este projeto um item indispensável no meu portfólio de estudos. Ele documenta com total transparência o meu progresso prático, a solidez das minhas bases de desenvolvimento front-end e o meu respeito pelo fluxo ideal de aprendizado: entender profundamente os fundamentos puros antes de avançar para as abstrações modernas do ecossistema de desenvolvimento web.
